# Reference Projects

Ardo keeps three small reference projects in the repository. They are maintained as product contracts: each is integration-tested, readable in isolation, and a better starting point for a human or coding agent than a generic generator template.

## Product Documentation

The [basic reference](https://github.com/sebastian-software/ardo/tree/main/examples/basic) is for one React app that publishes guides, Markdown/MDX pages, search, and static output.

## TypeScript Library Documentation

The [library reference](https://github.com/sebastian-software/ardo/tree/main/examples/library) pairs guide pages with TypeDoc output from a small TypeScript package.

## Monorepo Documentation

The [monorepo reference](https://github.com/sebastian-software/ardo/tree/main/examples/monorepo) keeps docs and several workspace packages in one repository.

Each reference README declares its intended shape, required files, configurable seams, generated-file boundary, and validation commands. When using an agent, link it to [Adopt Ardo in an Existing Project](/guide/adopt-ardo) first and ask it to select the closest reference before proposing changes.

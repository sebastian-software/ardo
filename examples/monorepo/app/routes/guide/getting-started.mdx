# Getting Started

This example demonstrates how to use Ardo to document multiple packages in a monorepo, each with its own TypeDoc entry point.

## How It Works

1. Two packages live in `packages/alpha/` and `packages/beta/`
2. Each has its own `tsconfig.json` and `src/index.ts`
3. A `prebuild.mjs` script runs `generateApiDocs()` twice — once per package
4. API docs are generated into `app/routes/api-reference/alpha/` and `app/routes/api-reference/beta/`

## Packages

### Alpha — Math Utilities

Functions for basic arithmetic: `add`, `subtract`, `multiply`, `divide`, and `clamp`.

### Beta — String Utilities

Functions for string manipulation: `capitalize`, `slugify`, `truncate`, and `repeat`.

## Project Structure

```
examples/monorepo/
├── packages/
│   ├── alpha/
│   │   ├── src/index.ts
│   │   └── tsconfig.json
│   └── beta/
│       ├── src/index.ts
│       └── tsconfig.json
├── app/
│   ├── routes/
│   │   ├── home.tsx
│   │   ├── guide/
│   │   │   └── getting-started.mdx
│   │   └── api-reference/    # Generated by TypeDoc
│   │       ├── alpha/
│   │       └── beta/
│   ├── root.tsx
│   ├── entry.client.tsx
│   └── entry.server.tsx
├── prebuild.mjs
├── vite.config.ts
└── package.json
```

## Running Locally

```bash
node prebuild.mjs
pnpm dev
```

## Building for Production

```bash
pnpm build
```

## Adding More Packages

To add a third package:

1. Create `packages/gamma/src/index.ts` with your TypeScript code
2. Add a `packages/gamma/tsconfig.json`
3. Add another `generateApiDocs()` call in `prebuild.mjs`
4. Add navigation links in `vite.config.ts` and `app/root.tsx`

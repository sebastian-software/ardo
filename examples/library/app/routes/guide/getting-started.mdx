# Getting Started

This example demonstrates how to use Ardo to document a TypeScript library with automatic API reference generation via TypeDoc.

## How It Works

1. Library source code lives in `src/index.ts`
2. A `prebuild.mjs` script runs `generateApiDocs()` from `ardo/typedoc` before the build
3. TypeDoc generates Markdown files into `app/routes/api-reference/`
4. React Router picks them up as routes automatically

## Project Structure

```
examples/library/
├── src/
│   └── index.ts              # Library source code
├── app/
│   ├── routes/
│   │   ├── home.tsx
│   │   ├── guide/
│   │   │   └── getting-started.mdx
│   │   └── api-reference/    # Generated by TypeDoc
│   ├── root.tsx
│   ├── entry.client.tsx
│   └── entry.server.tsx
├── prebuild.mjs              # TypeDoc generation script
├── tsconfig.api.json         # TypeScript config for TypeDoc
├── vite.config.ts
└── package.json
```

## Running Locally

For development, first generate the API docs, then start the dev server:

```bash
node prebuild.mjs
pnpm dev
```

## Building for Production

The build script runs prebuild automatically:

```bash
pnpm build
```

This executes `node prebuild.mjs && react-router build`.

## Adding Your Own Library

To adapt this example for your own library:

1. Replace the contents of `src/index.ts` with your library code
2. Update `tsconfig.api.json` if your entry points differ
3. Adjust `prebuild.mjs` for custom TypeDoc options
